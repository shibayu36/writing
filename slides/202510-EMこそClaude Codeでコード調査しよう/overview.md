読者と読後感
- EM。何らかの質問に答えるケースで、まずClaude Codeでコードも含めて素早く調査してみようと思える

開発チームを持つEMって、エンジニア以外から質問を受けることが多いですよね

- 今この機能ってどういう仕様だっけ？
- この機能を新しく作ろうとしたらどのくらい工数かかりそう？

回答するときどうする？（手を挙げてもらう）

- 詳しそうなエンジニアに聞く
- 自分でコードを読んでみる
- Claude Codeやcodexを活用して調査する

Claude Codeやcodexを活用して調査、が非常に役立ってるよって話

大前提: EMが回答する時、6~7割の精度での回答を素早くたくさんする事が多い

この前提に対して、コーディングエージェントによる調査がマッチする
コーディングエージェントは、完璧な回答は得られないが、6~7割くらいの精度でのコード調査ができるイメージ

最近の自分。とりあえず質問が投げられたら。例: 「ブログの予約投稿って、その時刻ぴったり投稿できるの？」

- まずClaude Codeで調査。知識を活用して調査しやすいプロンプトへ。「ブログの予約投稿はどのような仕組みで実装されている？」「予約投稿でどの程度遅れうる？」
- 出してくれたコードの場所を軽く眺めて正当性チェック。自分が正当性をチェックできないなら調査内容を持って他の詳しそうな人へヒアリング
- 内容を組み合わせて質問者に回答

より精度を上げるための工夫
- Thinkingモードで調査させよう
- 探索しやすいようにレポジトリ内アーキテクチャなどのドキュメントを充実させよう
    - https://github.com/shibayu36/Hibito/blob/main/CLAUDE.md#%E3%82%A2%E3%83%BC%E3%82%AD%E3%83%86%E3%82%AF%E3%83%81%E3%83%A3%E3%81%A8%E3%82%B3%E3%83%BC%E3%83%89%E6%A7%8B%E9%80%A0
- 調査用にプロンプトを整理しよう
```
## コード調査をお願いするときの指針
私からコード調査をお願いすることがある。その時はあなたに調査してもらいつつ、その妥当性を私も一緒にチェックしたい。
そこでコード調査をするときは結論だけを報告するのではなく、調査プロセスもリアルタイムに報告してほしい。
さらに結論をまとめるときは、結論に対する対する重要な証拠も一緒に提示して欲しい。
```
