最近、Claude Codeの以下の問題で非常に困っていた。
[https://x.com/shibayu36/status/2017804928992088292?s=20:embed:cite]

Claude Codeはコードを書かせるだけじゃなくて壁打ち役に使わせるのも便利だなと感じていた。そこでClaude Codeに手伝ってもらったらバグ報告も簡単にできるのではないかと試したところ https://github.com/anthropics/claude-code/issues/22315 のようにいい感じにissueを作れた。非常に体験が良かったので、参考のために今回はどんな感じに壁打ちしていったかを記録しておく。

まずClaude Codeはバグレポート用のテンプレがある。そのためそのテンプレを埋められるように情報を集める必要がある。このテンプレは.github/ISSUE_TEMPLATE/bug_report.ymlにある。Claude Codeのリポジトリをcloneしてその中でClaude Codeを起動し、そのテンプレを読み込ませてから開始した。

```
❯ バグレポートを送りたい。どんなバグかを僕にヒアリングして、類似issueも探索しながら作ろう。最終的なフォーマットは@.github/ISSUE_TEMPLATE/bug_report.yml を使う
```

続いて、具体的に起きていることをClaude Codeに教える。
```
❯ ctrl-bの前の文字に移動コマンドが効かなくなることがある。
  - 起動した直後は問題なく使える
  - C-rの履歴検索やC-gのエディタ起動で100%確定で動かなくなる
  - 一旦claudeを落とし、claude -cで復帰すると直る

本来はどんなケースでも前の文字に行けるCtrl-Bが動くことが期待されている。しかし特定の操作をするとエラーメッセージも特になく、Ctrl-Bを押しても反応しないという状況になる。
```

すると勝手に類似issueを探索してくれて、中身の詳細も確認しつつ、今回とは直接関係なさそうということも伝えてくれた。
```
⏺ おお、いくつか関連しそうなissueが見つかったよ！特に気になるのが：

  1. #21770 - [BUG] Ctrl-b is no longer non destructive backspace -
  まさにCtrl-B関連！
  2. #21519 - [BUG] Readline reverse-i-search (aka C-r) is broken - C-r関連

  詳細を確認してみるね！

  - #21519: C-r後にカーソルキーで編集できない問題

  あなたのバグは「C-rやC-gの後にCtrl-Bが効かなくなる」という状態変化の問題だから
  、新規issueの価値がありそう！

  duplicateで参照されてるissueも念のため確認するね。

⏺ 確認したissueはどれも違う問題だね：
  - #15422: C-r内の矢印キー問題

```


この結果、2.1.3でバグが入ったことを特定できた。最後にこれまでの情報を元にバグレポートをまとめさせて、手直ししてissue化できた。

### まとめ

Claude Codeと一緒に進めていくのは非常に体験が良かった。特に助かったのは以下の点。

- 類似issueを勝手に探して「これは別の問題だね」と判断してくれた
- bisectで試すバージョンを提案してくれて、結果を伝えると次のバージョンを出してくれた
- 最後にテンプレ通りの形式にまとめてくれた

大体30分くらいで今回のレポートが完成した。コードを書かせるだけじゃなく、こういう壁打ち相手としての使い方も非常に便利だということが再確認できた。
